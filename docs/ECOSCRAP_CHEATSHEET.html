üèóÔ∏è Project Overview & Architecture

What is EcoScrap?

EcoScrap is a sustainable scrap marketplace that connects customers with dealers to promote recycling and environmental conservation. The platform facilitates scrap pickup scheduling, product sales, and provides educational content.

Core Purpose: Enable sustainable scrap collection and recycling through a digital marketplace with role-based access control.

System Architecture

Backend Layer

  • Flask Web Framework
  • SQLAlchemy ORM
  • Flask-Login Authentication
  • Blueprint-based Routing

Database Layer

  • SQLite Database
  • 6 Core Models
  • Relational Design
  • Data Validation

Frontend Layer

  • HTML5 Semantic Markup
  • CSS3 with Custom Properties
  • ES6+ JavaScript Modules
  • Responsive Design

Technology Stack

Backend: Python 3.8+, Flask 2.3+, SQLAlchemy 2.0+, Flask-Login Database: SQLite (dev), PostgreSQL (production) Frontend: HTML5, CSS3, JavaScript ES6+ Security: Werkzeug, Flask-Login, Role-based Access Control Development: Black, Flake8, pytest, venv

üîÑ Application Workflows & Algorithms

User Registration & Onboarding

Registration Flow:
1. User fills registration form ‚Üí 2. Validate required fields ‚Üí 3. Check email uniqueness ‚Üí 4. Hash password ‚Üí 5. Create user account ‚Üí 6. Set default status (active) ‚Üí 7. Auto-login ‚Üí 8. Redirect to role dashboard

Product Approval Workflow

Product Lifecycle:
1. Dealer creates product (status: pending) ‚Üí 2. Admin reviews product ‚Üí 3. Admin approves/rejects ‚Üí 4. If approved: visible in marketplace ‚Üí 5. If rejected: hidden, dealer can resubmit ‚Üí 6. Customers can purchase approved products

Pickup Scheduling Algorithm

# Pickup Value Calculation Algorithm def calculate_pickup_value(material, weight): # 1. Fetch current rate for material rate = Rate.query.filter_by(material=material).first() # 2. Calculate estimated value estimated_value = rate.rate_per_kg * weight # 3. Apply any business rules (minimum weight, bonuses, etc.) if weight >= 50: # Bulk discount estimated_value *= 1.1 # 10% bonus # 4. Return calculated value return round(estimated_value, 2) # Pickup ID Generation Algorithm def generate_pickup_id(): timestamp = int(datetime.utcnow().timestamp()) return f"PU{timestamp}" # PU + Unix timestamp

Transaction Processing Flow

Order Processing:
1. Customer adds items to cart ‚Üí 2. Proceeds to checkout ‚Üí 3. Selects payment method ‚Üí 4. Creates transaction record ‚Üí 5. If COD: status = pending ‚Üí 6. If online: status = completed ‚Üí 7. Update product stock ‚Üí 8. Send confirmation ‚Üí 9. Dealer notification
